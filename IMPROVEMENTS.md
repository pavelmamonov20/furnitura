# Улучшения качества штрих-кодов

## Основные изменения в скрипте генерации штрих-кодов:

### 1. Повышение качества изображения:
- **module_height**: увеличена до 20.0 (было 15.0) — обеспечивает более высокие линии штрих-кода для лучшей читаемости
- **module_width**: увеличена до 0.55 (было 0.45) — делает линии штрих-кода немного толще для лучшей видимости
- **dpi**: установлено на 300 (было 1200, что было чрезмерно) — оптимальное качество для печати
- **quiet_zone**: увеличена до 1.0 (было 0.5) — обеспечивает большие поля по краям для лучшей читаемости сканерами

### 2. Улучшенные отступы:
- **top_padding**: 25 пикселей (было 10) — больше места сверху
- **bottom_padding**: 25 пикселей (было 0) — больше места снизу
- **left_padding**: 30 пикселей (было 0) — больше места слева
- **right_padding**: 30 пикселей (было 0) — больше места справа

### 3. Другие улучшения:
- **ROW_HEIGHT**: увеличена до 90 (было 85) — больше пространства для штрих-кодов
- Использование `Image.Resampling.LANCZOS` для лучшего алгоритма масштабирования
- Сохранение изображений с DPI 300 для высококачественной печати

### 4. Результаты:
- Штрих-коды стали более четкими и легко читаемыми
- Лучшая совместимость с различными сканерами
- Улучшенная визуальная презентация в Excel
- Повышенная надежность при печати и сканировании

Эти изменения обеспечивают значительно лучшее качество штрих-кодов по сравнению с оригинальной версией скрипта.
